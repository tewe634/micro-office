### 时间线

### 完成的功能

#### 1. 角色权限系统 (V2-V5 迁移)
- `sys_user` 新增 `role` 字段：ADMIN / HR / SALES / PURCHASE / FINANCE / STAFF
- JWT token 携带 role claim，JwtFilter 解析为 Spring Security authorities
- SecurityConfig 按角色限制 API 端点访问（/api/orgs, /api/users → HR+ADMIN; /api/admin → ADMIN; /api/objects → SALES+PURCHASE+FINANCE+ADMIN）
- `/api/users/me` 公共端点返回用户信息+菜单权限+对象类型权限
- 前端菜单按角色动态显示，路由守卫防 URL 直接访问

#### 2. 灵活权限配置 (V3-V5)
- `role_menu_permission` 表：角色级菜单权限（管理员可配置矩阵）
- `user_menu_permission` 表：个人级菜单权限覆盖（优先于角色默认）
- `position_object_type` 表：岗位级外部对象类型权限
- `user_object_type` 表：个人级对象类型权限覆盖（优先于岗位默认）
- 前端权限配置页面3个Tab：角色模块权限 / 人员权限（模块+对象类型） / 岗位对象类型

#### 3. 外部对象页面改造
- 改为 Tabs 切换，每个对象类型一个 Tab
- 按用户岗位/个人配置过滤可见类型
- 新增 org_id / owner_id 字段，支持归属到组织和负责人

#### 4. 个人门户（/portal）- 新公共模块，菜单最前面
- 基础信息 + 在职天数
- 同岗位排名（按完成工作数）
- 对内关系网（上下级）
- 对外关系网（关联的外部对象）
- 重大事件/奖励记录 CRUD（user_achievement 表，V6）

#### 5. 数据汇总（/dashboard）- 新公共模块，菜单最后面
- 时间维度：今天/昨天/本周/本月，统计工作数/节点数/评论数/打卡
- 组织维度：个人/部门/大区/公司，统计工作数+人员排名+外部对象分布

#### 6. 补齐的后端 CRUD API
- Thread: list(支持状态筛选), update, delete
- Comment: update, delete
- Clock: history(按天数查历史)
- Template: update; TemplateNode: update, delete
